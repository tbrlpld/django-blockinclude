{% extends '_test-base.html' %} {% load blockinclude %} {% block content %}

The only keyword on the blockinclude tag prevents the passing from other parent context, but content and the slot names will be available.

Notice how there won't be a footer, although the variable is defined in the parent. Passing the footer variable is prevented by the only keyword. The content and header variable, on the other hand, are available in the included template and rendered. They are not affected by the only keyword.

{% with footer="Adipisci atque aut incidunt" %} {% blockinclude 'includes/the-slotted-box.html' only %} {% slot "header" %} Phasellus nulla justo, dictum eu diam non, congue consequat ipsum. {% endslot %} Lorem ipsum dolor sit amet, consectetur adipiscing elit. {% endblockinclude %} {% endwith %}
{% endblock %}
