When more than one variable need should be filled with markup, blockinclude runs into the same issue as the default include tag. Single markup lines can be provided as a keyword argument after the `with` keyword in the tag. But, multi-line markup or even template logic would not work.

This is where the slot tag comes it. Inside of blockinclude, you wrap a block of markup or template logic in a slot/endslot block. The template logic is executed in the parent and the resulting markup is passed to the included template with the defined name of the slot.

{% blockinclude 'includes/the-slotted-box.html' %} {% slot header %} Phasellus nulla justo, dictum eu diam non, congue consequat ipsum. {% endslot %} Lorem ipsum dolor sit amet, consectetur adipiscing elit. {% slot footer %}
    {% for item in items %}
  • {{ item }}
    • {% endfor %}
{% endslot %} {% endblockinclude %}
