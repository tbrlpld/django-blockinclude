When more than one variable needs to be filled with markup, blockinclude runs into the same issue as the default include tag. Single markup lines can be provided as a keyword argument after the `with` keyword in the tag. But, multi-line markup or even template logic would not work.

This is where the slot tag comes in. Inside of blockinclude, you wrap a block of markup in a slot/endslot block. The markup in the slot block is available in the included template as a variable with the name of the slot. All other markup in the blockinclude, that is outside the slot block, is still passed as the content variable.

{% blockinclude 'includes/the-slotted-box.html' %} {% slot header %} Phasellus nulla justo, dictum eu diam non, congue consequat ipsum. {% endslot %} Lorem ipsum dolor sit amet, consectetur adipiscing elit. {% endblockinclude %}
{% endblock %}
